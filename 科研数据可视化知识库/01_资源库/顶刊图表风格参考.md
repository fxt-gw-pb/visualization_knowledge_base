# 顶刊图表风格参考

> 目标不是“复制顶刊同款颜色/图”，而是**提炼它们背后的设计原则**，转成本库可执行规则。
> 任何“同款”冲动，先回到这里问：这条原则我学到了吗？

## 1. 顶刊图表共有的设计原则

| 原则 | 含义 | 在本库的落点 |
|---|---|---|
| 简洁 | 去掉一切不传达数据的元素（3D、阴影、重网格、冗余边框）| [[图表返修规则总表]] 装饰类 |
| 高信息密度 | 一张图尽量多传达信息，但不堆砌；small multiples | [[分面与多面板设计]] |
| 明确轴标签 | 轴有量纲单位、刻度可读、必要时 log 标注 | [[坐标轴设计]] |
| 控制颜色数量 | 分类色少而稳定，语义一致；连续/发散用感知均匀色 | [[配色系统总览]] |
| 多面板结构清晰 | A/B/C/D 标注、对齐、共享图例 | [[多面板组合图_MultiPanel]] |
| 注释服务结论 | 标注只点关键比较/关键点，不喧宾夺主 | [[标签与注释设计]] |
| 不做无意义装饰 | 数据墨水比高（Tufte）| [[ggplot2通用主题]] / [[matplotlib通用主题]] |
| 可复现 | 图与代码/数据绑定 | [[R导出规范]] / [[Python导出规范]] |

## 2. 各家“气质”速记（仅作风格参考，非硬规则）

> 以下是公开可观察的总体倾向，不同文章差异很大；**以投稿期刊的 author guideline 为准**。

| 期刊家族 | 总体气质 | 可借鉴点 |
|---|---|---|
| Nature 系 | 紧凑、无衬线、面板密、配色克制 | 单栏密排、A/B/C/D 一致、字号紧凑 |
| Science / AAAS | 信息密度高、面板组合复杂 | 多面板叙事、图注详尽 |
| Cell 系 | 图形化叙事强、graphical abstract | 用结构图讲机制（非统计图） |
| Lancet / NEJM / JAMA（医学）| 克制、临床导向、森林图/KM/CONSORT 多 | 森林图表图对齐、KM+risk table、流程图规范 |

医学三大刊（Lancet/NEJM/JAMA）最该学的不是颜色，而是：

- **森林图**的左表右图对齐、参考线、合并效应（[[森林图_ForestPlot]]）。
- **KM 曲线** 必带 risk table 与删失标记（[[KM生存曲线_KaplanMeier]]）。
- **CONSORT/入组流程图** 的规范化（规划中卡片）。
- **Table 1（baseline characteristics）** 的可视化呈现（[[多面板组合图_MultiPanel]] Figure 1 风格）。

## 3. 期刊投稿的硬性技术要求（普遍共性）

```text
矢量优先：线图/统计图 → PDF/EPS/SVG（可无损缩放、嵌字体）
栅格 dpi：线图 ≥ 600–1200 dpi；含照片/热图 ≥ 300 dpi（TIFF 常见）
颜色模式：印刷可能要求 CMYK；屏幕/在线 RGB（多数现代期刊接受 RGB）
字体：嵌入字体；正文图字号通常 7–9 pt（终稿尺寸下可读）
尺寸：单栏 ~85–90mm / 双栏 ~170–180mm（见 [[论文单栏双栏尺寸]]）
线宽：≥ 0.5–1 pt，避免缩放后消失
```

> 具体阈值各刊不同，投稿前查该刊 “Figure preparation / Artwork guidelines”。

## 4. 把顶刊原则转成自动检查（给 Agent）

每张图出来后，问这 7 句（对应 [[科研图表质量检查清单]]）：

1. 有没有非数据装饰可以删？
2. 颜色数量是否 ≤ 必要？语义一致吗？
3. 每个轴都有单位/清楚刻度吗？
4. 多面板对齐、字号统一、有 A/B/C/D 吗？
5. 注释是否只服务关键结论？
6. 误差度量（CI/SE/SD）写清了吗？
7. 导出是不是矢量 + 正确尺寸？
