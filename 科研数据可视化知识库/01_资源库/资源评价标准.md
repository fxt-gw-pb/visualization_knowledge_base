# 资源评价标准

> 决定一个绘图资源能否进本库、放在“核心”还是“参考”的统一尺子。避免盲目纳入过时/维护差/license 不明的包。

## 1. 评分维度

| 维度 | 看什么 | 权重 |
|---|---|---|
| 维护状态 | 最近提交/发布、issue 响应、是否归档 | 高 |
| 文档质量 | 是否有完整文档站 + 可运行示例 | 高 |
| license | 是否开放且明确（MIT/BSD/GPL/Apache）| 高（硬门槛）|
| 生态契合 | 是否融入 tidyverse / matplotlib / sklearn 生态 | 中 |
| 科研适配 | 是否支持统计语义（CI、p、生存、注释）| 高 |
| 稳定性 | API 是否频繁破坏性变更 | 中 |
| 可复现 | 是否易固定版本、依赖是否重 | 中 |

## 2. 适合程度判定

```text
高：维护活跃 + 文档全 + license 明确 + 科研常用 → 进核心，写进卡片主路径
中：可用但有短板（维护慢/依赖重/定制弱）→ 作为补充或快速方案
低：过时/维护停滞/license 不明/API 混乱 → 不纳入核心，仅在“注意”里提示
```

## 3. 必须明确记录的字段（资源卡片统一模板）

每个资源在 [[R绘图资源索引]]/[[Python绘图资源索引]]/[[科研配色资源索引]] 里至少记录：

```markdown
# 资源名称
## 基本信息
- 名称：
- 语言：R / Python / 通用
- 官网/仓库：
- 文档：
- GitHub：
- 维护状态：活跃 / 缓慢 / 停滞 / 归档
- license：
- 适合程度：高 / 中 / 低
## 主要用途
（解决什么绘图问题）
## 适合的图表类型
## 不适合的场景
## 可迁移到本知识库的内容
（图表分类 / 代码模板 / 配色 / 主题 / 参数 / 示例图架构）
## 代表性代码/思路
（关键用法，不大段复制文档）
## 参考链接
```

## 4. 红旗清单（出现即降级或拒收）

- 🚩 GitHub 已 archived 或 2+ 年无提交（除非是稳定到无需更新的小工具）。
- 🚩 license 缺失/不明 → 不纳入核心，代码示例也不照抄。
- 🚩 文档只有 README、无可运行示例。
- 🚩 频繁破坏性更新且无迁移指南。
- 🚩 依赖一堆冷门未维护包，复现困难。

## 5. 图片/示例素材的 license 规则（与 `07_示例图像库` 一致）

```text
license 明确开放（CC/BSD/MIT…）→ 可下载缩略图 + 注明来源
license 受限 / 不明 / 顶刊图    → 只记链接 + 文字描述，不保存图片
任何情况都要记录：原始链接、来源、作者/机构、license、抓取日期
```

## 6. 复审节奏

- 每次大版本迭代（如新增一族卡片）顺手复审核心资源是否仍活跃。
- 发现资源过时/迁移，在对应索引条目标注 `⚠️ 2026-xx 状态变更`，并在 [[数据可视化知识库总览]] 的“发现的问题”里记一笔。
