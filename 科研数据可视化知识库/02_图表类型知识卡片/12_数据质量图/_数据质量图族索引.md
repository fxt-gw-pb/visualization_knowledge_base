# 12 数据质量图族索引

> 面向 **EHR / 真实世界数据(RWD)** 的数据质量与探索（EDA）：先把“数据本身可不可信”看清楚，再做分析。建模/出结果图之前的前置步骤。

## 成员

| 图表 | 卡片 | 给什么信息 | 状态 |
|---|---|---|---|
| 缺失数据模式图 Missingness | [[缺失数据图_Missingness]] | 每变量缺失率 + 缺失模式 | tested |
| 共缺热图 Co-missingness | ⬜ planned | 变量×变量缺失相关 | planned |
| 异常值/离群图 Outlier | ⬜ planned | 数值分布异常/录入错误 | planned |
| 数据完整度时间图 | ⬜ planned | 缺失随随访期/时间变化 | planned |

## 选型速记

```text
先看每列缺多少 + 缺得有无规律   → 缺失数据模式图（本族首图）
看哪些变量成套一起缺           → 共缺热图（规划中）
看数值有无异常/录入错误         → 离群图（规划中）
看缺失随时间/随访变化           → 数据完整度时间图（规划中）
```

## 共性要点
- RWD 分析**第一步**：缺失/异常先看清，决定删除 vs 填补 vs 专门建模。
- 缺失/异常处理方式必须在方法里交代，影响所有下游结论。
- 与 [[CONSORT流程图_Consort]] 联动：因数据质量排除的记录计入流程图。
