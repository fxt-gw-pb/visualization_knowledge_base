# 分类变量配色

> 适用：**无序类别**——性别、病例/对照、治疗/对照、风险分组、多类别分组。
> 一句话：**少而稳、色盲友好、语义一致。**

## 1. 适用数据

- 性别（SEX）、吸烟状态（CURSMOKE）、是否高血压（PREVHYP）、随访期（PERIOD）、年龄组（AGE_group）……
- 凡是“几个互不相干的桶”，且类别之间**无大小顺序**。

## 2. 默认方案：Okabe-Ito（色盲安全 8 色）

```text
#E69F00 橙   #56B4E9 天蓝   #009E73 绿   #F0E442 黄
#0072B2 蓝   #D55E00 朱红   #CC79A7 粉   #999999 灰（/#000000 黑）
```

- 这是分类色的**事实标准**，红绿色盲也能区分。
- registry 名：`cat_main`（见 [[配色系统总览]]）。
- R 4.0+：`palette.colors(palette = "Okabe-Ito")`。

## 3. 备选方案

| 方案 | 何时用 | 接入 |
|---|---|---|
| ColorBrewer `Set2`/`Dark2` | 想要更柔和/更饱和；≤8 类 | R `scale_*_brewer(palette="Set2")`；Py `palette="Set2"` |
| ggsci 期刊色（NPG 等）| 想要期刊气质 | R `scale_color_npg()`；**先过色盲检查** |
| 灰阶 + 形状/线型 | 黑白打印、类别少 | 见 [[色盲友好与打印友好原则]] |

## 4. 颜色数量规则

- **≤ 8** 个主色；绝对上限 ~10。
- 超过上限的处理（也是返修规则）：
  1. **分面**（facet）把类别拆到子图；
  2. **合并低频类别**为 "Other"；
  3. **加形状/线型**辅助（颜色 + shape 双编码）；
  4. 若本质是有序的，改用顺序色（见 [[连续变量配色]]）。

## 5. 语义一致性（重要）

- 同一概念全 figure / 全文用同一颜色：**病例永远朱红、对照永远蓝**，不要这图红那图绿。
- 二分类医学语义直接用固定映射 → [[医学二分类配色]]。
- 颜色别和“好/坏”乱挂钩；需要方向语义时用发散色 → [[发散变量配色]]。

## 6. 代码片段

**R / ggplot2**

```r
okabe_ito <- c("#E69F00","#56B4E9","#009E73","#F0E442",
               "#0072B2","#D55E00","#CC79A7","#999999")
ggplot(df, aes(x, y, fill = group)) +
  geom_col() +
  scale_fill_manual(values = okabe_ito) +   # >>> PARAM palette = cat_main
  theme_minimal()
```

**Python / seaborn**

```python
okabe_ito = ["#E69F00","#56B4E9","#009E73","#F0E442",
             "#0072B2","#D55E00","#CC79A7","#999999"]
sns.barplot(data=df, x="x", y="y", hue="group", palette=okabe_ito)  # >>> PARAM
```

## 7. 常见错误

- ❌ 类别太多导致相邻色难分 → 分面/合并/加形状。
- ❌ 用红配绿编码两组（色盲不友好）→ 改红/蓝。
- ❌ 同一概念换色导致跨图不一致。
- ❌ 把有序类别（如年龄组 1<2<3）用无序分类色 → 改顺序色更达意。
