# 发散变量配色

> 适用：**有明确中心、有正负方向的量**——fold change、回归系数、z-score、标准化残差、相关系数、效应差值。
> 一句话：**中心必须对准 0（或参考值），正负颜色语义固定、不可随意反转。**

## 1. 适用数据

- log2 fold change（火山图/热图）。
- 回归系数 β、效应方向（[[模型效应方向配色]]）。
- z-score 标准化的表达矩阵（热图）。
- 相关矩阵（−1 ~ +1，中心 0）。
- 标准化残差、与参考值的偏差。

## 2. 默认方案：RdBu（对称、中心白）

- registry 名：`div_rdbu`（见 [[配色系统总览]]）。
- 红=正、蓝=负、白=中心（0）。`RdBu` 默认蓝在正端，常用 `RdBu_r`（reverse）让红表示正/上调。
- 论文级替代：Crameri `vik`（`div_vik`），感知均匀 + 色盲安全。

## 3. 硬规则

1. **中心对齐**：色阶中点必须落在数据的中心值（通常 0）。
   - matplotlib 用 `TwoSlopeNorm(vcenter=0)`；ggplot2 用 `scale_*_gradient2(midpoint=0)`。
2. **对称范围**：`vmin = -max(|x|)`, `vmax = +max(|x|)`，否则白色不在 0，产生误导。
3. **方向语义固定且全文一致**：一旦“红=上调/正效应”，全 figure / 全文不变。
4. **不要随意反转含义**（红=好绿=坏之类）→ 这是 [[图表返修规则总表]] 的红线之一。
5. 红绿发散对色盲不友好 → 优先**红-蓝**或 Crameri `vik`/`broc`。

## 4. 代码片段

**R / ggplot2**

```r
ggplot(df, aes(x, y, fill = logFC)) +
  geom_tile() +
  scale_fill_gradient2(low = "#2166AC", mid = "white", high = "#B2182B",
                       midpoint = 0,                       # >>> PARAM center
                       limits = c(-max(abs(df$logFC)), max(abs(df$logFC)))) +
  theme_minimal()
# 感知均匀替代： scico::scale_fill_scico(palette="vik", midpoint=0)
```

**Python / matplotlib**

```python
from matplotlib.colors import TwoSlopeNorm
m = max(abs(df.logFC))
norm = TwoSlopeNorm(vmin=-m, vcenter=0, vmax=m)   # >>> PARAM center=0, symmetric
plt.scatter(df.x, df.y, c=df.logFC, cmap="RdBu_r", norm=norm)
plt.colorbar(label="log2 fold change")
# seaborn 热图： sns.heatmap(corr, cmap="RdBu_r", center=0, vmin=-1, vmax=1)
```

## 5. 常见错误

- ❌ 用顺序色（viridis）画有正负的量 → 看不出方向，0 不突出。
- ❌ 中心不在 0（范围不对称）→ 白色偏移，夸大一侧。
- ❌ 红绿发散 → 色盲读不出。
- ❌ 不同图里红/蓝含义互换。

## 6. 与顺序色的区别

- 只有低→高、无中心 → 顺序色，见 [[连续变量配色]]。
- 有中心 + 正负 → 发散色（本页）。
