# 连续变量配色

> 适用：**单向有序的量**——风险评分、表达量、预测概率、年龄、血压、BMI、密度。
> 一句话：**亮度单调、感知均匀、永远别用彩虹/jet。**

## 1. 适用数据

- 概率（0→1）、计数密度、表达量、连续临床指标（BMI/TOTCHOL/GLUCOSE）映射到颜色时。
- 热图的连续值、散点的连续第三维（color = 某连续量）。

## 2. 默认方案：viridis

- registry 名：`seq_viridis`（见 [[配色系统总览]]）。
- 优点：**感知均匀**（等数值差→等视觉差）、**色盲友好**、**灰度打印仍单调**。
- 家族：`viridis`（紫→蓝→绿→黄）、`magma`/`inferno`/`plasma`（暖向）、`cividis`（最强色盲优化）。

## 3. 备选方案

| 方案 | 何时用 | 接入 |
|---|---|---|
| ColorBrewer Sequential（Blues/Greens/Oranges…）| 想要单一色相由浅到深；勾选 colorblind-safe | R `scale_*_distiller(palette="Blues")`；Py `cmap='Blues'` |
| Crameri 顺序（batlow/lajolla）| 论文级、严格感知均匀 | R `scico`；Py `cmcrameri` |
| cividis | 强调色盲安全 | `cmap='cividis'` |

## 4. 硬规则

1. **避免 rainbow / jet / hsv**：亮度非单调、制造伪边界、灰度失真（[[色盲友好与打印友好原则]] 有反例）。
2. **亮度应随数值单调变化**：低值浅/暗、高值深/亮（同一方向）。
3. 离群值会“吃掉”整个色阶 → 用**分位映射**或**截断**（clip）色阶范围（也是返修规则）。
4. 连续量**不要分太多档**；若必须离散化，≤ 5–7 档并标注分界。

## 5. 代码片段

**R / ggplot2**

```r
ggplot(df, aes(x, y, color = value)) +
  geom_point() +
  scale_color_viridis_c(option = "viridis") +   # >>> PARAM seq palette
  theme_minimal()
# 热图连续填充： + scale_fill_viridis_c()
```

**Python / matplotlib + seaborn**

```python
plt.scatter(df.x, df.y, c=df.value, cmap="viridis")   # >>> PARAM
plt.colorbar(label="value")
# 截断离群： vmin/vmax = df.value.quantile([.02,.98])
sns.heatmap(mat, cmap="viridis")
```

## 6. 常见错误

- ❌ jet/rainbow 显得“丰富”，实则误导 + 色盲不友好。
- ❌ 用发散色编码纯单向量（无意义的“中心”）→ 改顺序色。
- ❌ 色阶被极端值拉爆，主体一片同色 → 分位/截断/对数。
- ❌ 连续量硬塞进分类色板。

## 7. 与发散色的区别

- 没有“中心/参考值”，只有低→高 → **顺序色（本页）**。
- 有正负或中心（0、均值、对角线）→ **发散色**，见 [[发散变量配色]]。
