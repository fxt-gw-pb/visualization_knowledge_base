# 医学二分类配色

> 医学/流行病学里反复出现“两组”对照。这里给**统一语义 + 推荐固定色**，保证全库一致、跨图可读、可复用进 Skill。
> 全部基于 Okabe-Ito，色盲友好。

## 1. 统一语义映射表（registry）

| 语义对 | 突出组（暖/亮）| 参照组（冷/灰）| registry 名 | 备注 |
|---|---|---|---|---|
| case / control（病例/对照）| 病例 `#D55E00` 朱红 | 对照 `#0072B2` 蓝 | `med_case_control` | 病例暖、对照冷 |
| disease / no disease | 患病 `#D55E00` | 未患 `#0072B2` | 同上 | |
| treatment / control（治疗/对照）| 治疗 `#009E73` 绿 | 对照 `#999999` 灰 | `med_treat_control` | 治疗突出、对照中性 |
| exposed / unexposed（暴露/未暴露）| 暴露 `#E69F00` 橙 | 未暴露 `#56B4E9` 天蓝 | `med_exposed` | |
| high risk / low risk | 高危 `#D55E00` | 低危 `#0072B2` | `med_risk2` | 三档见下 |

风险三档（如需要）：高 `#D55E00` / 中 `#E69F00` / 低 `#0072B2`（暖=高危，语义直觉）。

## 2. 选色原则

1. **突出组用暖色/饱和色，参照组用冷色/中性色**——让读者一眼抓到“关注的那组”。
2. **避免红-绿对立**编码疾病/暴露（色盲不友好）。需要时红-蓝。
3. **治疗 vs 对照**常让对照用灰，强调“干预组才是主角”。
4. 方向有“好坏”含义时，**不要把不利组配成讨喜的绿色**（[[图表返修规则总表]] 红线）。
5. 全文一致：一篇论文里“病例”永远朱红。

## 3. 在本库 Framingham 数据中的落地

| 变量 | 取值 | 建议映射 |
|---|---|---|
| SEX | 0=Male / 1=Female（已确认）| 中性两色：Male `#0072B2` / Female `#D55E00`（性别无“好坏”，对等不暗示危险）|
| CURSMOKE | 0 不吸 / 1 吸烟 | 暴露语义 → 吸烟 `#D55E00`、不吸 `#0072B2` |
| PREVHYP | 0 无 / 1 有高血压 | 疾病语义 → 有 `#D55E00`、无 `#0072B2` |
| DEATH | 0 存活 / 1 死亡 | 结局语义 → 死亡 `#D55E00`、存活 `#0072B2`（生存图里见 [[KM生存曲线_KaplanMeier]]）|

> 注意：性别没有“突出/参照”之分，用两个**对等**的中性色，别让一种性别显得“更危险”。

## 4. 代码片段

**R**

```r
med_case_control <- c(control = "#0072B2", case = "#D55E00")  # >>> PARAM registry
ggplot(df, aes(x, fill = factor(PREVHYP, labels=c("No","Yes")))) +
  geom_bar(position="fill") +
  scale_fill_manual(values = unname(med_case_control), name = "Hypertension")
```

**Python**

```python
med = {"No": "#0072B2", "Yes": "#D55E00"}   # >>> PARAM
sns.boxplot(data=df, x="PREVHYP_label", y="BMI",
            palette=med, order=["No","Yes"])
```

## 5. 常见错误

- ❌ 病例/对照颜色逐图变化。
- ❌ 红绿编码疾病。
- ❌ 给“死亡/患病”配讨喜颜色，弱化结论严重性。
- ❌ 给无序的性别强行配“危险色”。
