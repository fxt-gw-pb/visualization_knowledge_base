# 配色系统总览

> 核心思想：**不追求“顶刊同款颜色”，而是建立按数据语义选色的 publication-style palette registry。**
> 一切配色先问一句：**这组数据是分类、连续、还是发散？** 答完就知道该去哪张子表。

## 1. 选色决策树

```text
你的颜色在编码什么？
├─ 类别（无序，如性别/分组）       → 分类色 → [[分类变量配色]]
│    └─ 恰好两类且有医学语义？      → [[医学二分类配色]]
├─ 一个连续量（有序，低→高）       → 顺序色 → [[连续变量配色]]
└─ 有正负/中心的量（fold change/系数/z）→ 发散色 → [[发散变量配色]]
                                      └─ 编码“效应方向” → [[模型效应方向配色]]
```

三条铁律（所有子表都遵守）：

1. **语义匹配**：分类数据别用连续色，连续数据别用分类色，发散数据必须有中心。
2. **色盲友好 + 打印友好**：默认就要过检查（见 [[色盲友好与打印友好原则]]）。
3. **稳定一致**：同一概念（如“病例 vs 对照”）在全库/全 figure 用**同一组**颜色。

## 2. 命名色板 Registry（机器可读）

> 卡片与代码**只引用名字**，不写死 hex；改色只改这里一处。Agent 注入时按名取值。

### 2.1 分类（qualitative）

| 名字 | 用途 | 色值（hex）| 来源 |
|---|---|---|---|
| `cat_main` | 通用分类（≤8 类）默认 | `#E69F00 #56B4E9 #009E73 #F0E442 #0072B2 #D55E00 #CC79A7 #999999` | Okabe-Ito |
| `cat_brewer_set2` | 柔和分类（≤8）| ColorBrewer `Set2` | ColorBrewer |
| `cat_npg` | 期刊风分类 | ggsci NPG（用前过色盲检查）| ggsci |

### 2.2 顺序（sequential，连续量低→高）

| 名字 | 用途 | 来源 |
|---|---|---|
| `seq_viridis` | 默认连续（感知均匀+色盲安全）| viridis |
| `seq_cividis` | 强色盲优化连续 | viridis::cividis |
| `seq_blues` | 单色顺序（如概率/密度）| ColorBrewer Blues |
| `seq_batlow` | 论文级感知均匀顺序 | Crameri batlow |

### 2.3 发散（diverging，有中心 0）

| 名字 | 用途 | 中心 | 来源 |
|---|---|---|---|
| `div_rdbu` | 通用正负（红正/蓝负，可反）| 0 | ColorBrewer RdBu |
| `div_vik` | 论文级感知均匀发散 | 0 | Crameri vik |
| `div_coolwarm` | 蓝-白-红（matplotlib）| 0 | matplotlib coolwarm |

### 2.4 医学二分类语义（固定映射）

| 语义对 | 高风险/病例/暴露/处理 | 低风险/对照/未暴露 | 备注 |
|---|---|---|---|
| `med_case_control` | `#D55E00`(朱红) | `#0072B2`(蓝) | 病例暖色、对照冷色；避免红绿 |
| `med_treat_control` | `#009E73`(绿，处理) | `#999999`(灰，对照) | 处理突出、对照中性 |
| `med_risk` | `#D55E00` high / `#E69F00` mid / `#0072B2` low | — | 风险三档，暖=高危 |

详见 [[医学二分类配色]]。

## 3. 颜色数量上限

| 通道 | 上限 | 超了怎么办 |
|---|---|---|
| 分类填充/描边 | 8（绝对上限 ~10）| 分面 / 合并低频为 "Other" / 加形状/线型辅助 |
| 连续色阶 | 不分级（渐变）| 若需分级 ≤ 5–7 档 |
| 发散色阶 | 对称分级 ≤ 9 | 保持中心档为浅色/白 |

## 4. R / Python 接入速查

| 语义 | R（ggplot2）| Python |
|---|---|---|
| 分类 Okabe-Ito | `scale_*_manual(values=okabe_ito)` 或 `palette.colors(palette="Okabe-Ito")` | `sns.set_palette([...])` / `palette=[...]` |
| 分类期刊 | `scale_color_npg()`（ggsci）| 手抄 hex 列表 |
| 顺序 viridis | `scale_*_viridis_c()` | `cmap='viridis'` |
| 发散 RdBu | `scale_*_distiller(palette="RdBu")` / `scale_*_gradient2()` | `cmap='RdBu_r'`, 设 `vcenter=0`（`TwoSlopeNorm`）|
| Crameri | `scico::scale_*_scico(palette="vik")` | `cmcrameri` `cmap=cmc.vik` |

## 5. 与其他系统的关系

- 每张图表卡片第 6 节“配色选择”都引用本系统，不重复定义颜色。
- 收尾质检看 [[科研图表质量检查清单]] 的 C 区“颜色是否色盲友好/数量受控”。
- 发散色不可随意反转（红=好绿=坏之类），见 [[模型效应方向配色]]。
