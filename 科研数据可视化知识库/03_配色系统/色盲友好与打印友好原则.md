# 色盲友好与打印友好原则

> 约 8% 男性、0.5% 女性有色觉缺陷（多为红绿）。论文还常被黑白打印/复印。配色默认就要扛住这两关。
> 这是所有配色子表的“底线检查”。

## 1. 色盲友好（colorblind-safe）

### 原则

1. **不要用红-绿对立**编码关键语义（最常见的红绿色盲读不出）。
2. **不靠颜色单一通道**传达信息——叠加**形状、线型、直接标注、亮度差**做冗余编码（redundant encoding）。
3. 分类首选 **Okabe-Ito**；连续首选 **viridis/cividis**；发散首选 **红-蓝（RdBu）或 Crameri vik**。
4. 相邻类别保证**亮度/明度有差异**，而不只是色相差异。

### 检查方法

| 方法 | 工具 |
|---|---|
| 模拟色盲看图 | Coblis、Viz Palette（见 [[科研配色资源索引]]）|
| 代码内模拟 | R `colorspace::deutan()/protan()/tritan()`；Python `colorspacious` |
| 灰度转换看是否仍可分 | 见下方“打印友好” |

## 2. 打印友好（print / photocopy-safe）

### 原则

1. 关键区分在**转成灰度后仍能分辨**——即亮度（明度）要有梯度，不能只靠色相。
2. 连续色用**亮度单调**的色图（viridis/Crameri），灰度打印仍单调；**rainbow/jet 灰度后乱套**。
3. 线图叠加**线型**（实线/虚线/点线）作为颜色的备份。
4. 填充图可叠加**纹理/网格密度**（谨慎，过多反而乱）。
5. ColorBrewer 直接标了 “print friendly / photocopy safe”，按它筛。

### 快速自检

```text
把图转灰度（去色）：
- 各组还能分清吗？  不能 → 加形状/线型/标注
- 连续色阶还单调吗？ 不单调 → 换 viridis/Crameri
```

R：`ggsave` 后 `magick::image_convert(colorspace="gray")` 预览；
Python：`plt.imsave` 后用 `cv2.cvtColor(..., COLOR_RGB2GRAY)` 或截图转灰度看。

## 3. 反例（不要用）

| 反例 | 问题 | 改法 |
|---|---|---|
| rainbow / jet 连续色 | 亮度非单调、伪边界、灰度失真 | viridis / Crameri |
| 红配绿两组 | 红绿色盲读不出 | 红配蓝 / Okabe-Ito |
| 只用颜色区分多条线 | 色盲 + 灰度都难 | 颜色 + 线型 + 直接标注 |
| 浅黄配白 | 对比不足，投影/打印看不清 | 提高明度对比 |
| 8 种高饱和分类色铺满 | 视觉疲劳 + 难分 | 减类别 / 分面 / Okabe-Ito |

## 4. 冗余编码示例

**R**

```r
ggplot(df, aes(time, value, color = group, linetype = group, shape = group)) +
  geom_line() + geom_point() +
  scale_color_manual(values = okabe_ito)   # 颜色+线型+形状三重冗余
```

**Python**

```python
styles = {"A": ("#0072B2","-","o"), "B": ("#D55E00","--","s")}
for g,(c,ls,mk) in styles.items():
    s = df[df.group==g]
    plt.plot(s.time, s.value, color=c, linestyle=ls, marker=mk, label=g)
```
