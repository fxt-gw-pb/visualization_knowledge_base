# Agent 调用流程

> 未来 Skill 被调用时，Agent 内部走的步骤。把 [[作图Skill总体设计]] 落成一条可执行流水线。

## 1. 主流程（端到端）

```text
[用户请求]
  数据 + 研究问题 + 用途（论文/汇报/探索）
   ↓
1. 解析意图 & 探查数据
   - 识别变量类型（连续/分类/二分类/时间-事件）
   - 缺失/分布/样本量快检（参考 [[变量字典]] 思路）
   ↓
2. 图表选型（[[图表决策流程]]）
   - data_type + 问题 → 候选图（读卡片 frontmatter）
   - 给出主选 + 备选 + 理由
   ↓
3. 后端路由（[[R与Python后端选择规则]]）
   - 数据所在管道 + 图“天花板” + 下游用途 → R / Python
   ↓
4. 取模板 & 注入参数（[[输入输出规范]]）
   - 读卡片第 7/8 节代码模板
   - 填 # >>> PARAM（x/y/分组/配色 registry/分面/尺寸）
   - 配色查 [[配色系统总览]]，尺寸查 [[论文单栏双栏尺寸]]
   ↓
5. 生成可运行代码（不黑盒）
   - 输出 .R / .py 脚本 + 注释 + 导出语句
   ↓
6. 执行出图（环境允许时）
   ↓
7. 自检（[[科研图表质量检查清单]]）
   - A 语义 / B 统计 / C 视觉 / D 导出 打分
   ↓
8. 自动返修（[[图表返修规则总表]] IF→THEN）
   - 标签重叠/图例过长/overplot/p值过多… 改参数重画
   - 记录到“迭代记录”
   ↓
9. 导出（[[R导出规范]] / [[Python导出规范]]）
   - 论文 PDF/SVG；汇报 PNG+PDF；规范命名
   ↓
[交付：图 + 可复现代码 + 质检报告 + 选型理由]
```

## 2. 决策点上的“问用户 vs 自动决定”

| 决策 | 默认自动 | 何时问用户 |
|---|---|---|
| 图表类型 | 自动选主选 | 多个等价候选且影响结论时 |
| 后端 | 自动按规则 | 用户明确偏好或环境受限 |
| 配色 | 自动按语义 registry | 有品牌/期刊指定色 |
| 用途/尺寸 | 默认论文 | 请求里没说用途时确认一次 |
| 统计方法（p 值检验等）| 用安全默认（非参） | 涉及推断严谨性时提示用户确认 |

## 3. 失败/兜底

- 数据类型识别不确定 → 让用户确认变量角色。
- 选型有歧义 → 给 2 个候选 + 各自适用/不适用（引卡片第 2/3 节）。
- 环境缺包 → 输出代码 + 安装指引，不假装出图。
- 质检不过且自动返修无效 → 报告问题，给人工建议。

## 4. 与质量门的关系

```text
出图 → 质检得分 < 75 → 必返修
       75–89 → 小修
       ≥ 90 → 通过，可标记接近 publication_ready
```
（口径见 [[科研图表质量检查清单]] 评分版）

## 5. 日志与可追溯

每次调用记录：输入 → 选型理由 → 后端 → 参数 → 质检分 → 返修动作 → 输出文件。
对应卡片第 14 节“迭代记录”的版本号，便于回溯。
