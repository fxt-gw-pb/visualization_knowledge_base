# MVP 路线图

> 从“知识库”到“可调用作图 Skill”的分阶段路线。每阶段有明确产出与“完成定义（DoD）”。

> [!note] 实现进展（2026-06-10 追加）
> - 阶段 0、1 **已完成**（12 卡片全 `tested`，R/Python 各 14 张实测图）。
> - 阶段 2 已有 **MVP**：`registry/charts.json`（由 `scripts/build_registry.py` 从 frontmatter 派生）即机器可读 spec；配色 registry 已命名化。
> - 阶段 3、4 已有 **agent 驱动 MVP**：Claude Code Skill **`sci-plot`**（`~/.claude/skills/sci-plot/`）把「选型→后端路由→模板注参→出图→质检→返修」落成可执行流水线，读本 KB 而非硬编码。键入 `/sci-plot` 或直接描述作图需求即可触发。
> - 仍待做的是把阶段 3 从 agent 推理升级为**确定性 spec→代码渲染器**，以及阶段 5 回归测试 + 扩展图表族。

## 阶段 0：知识库地基 ✅（2026-06-10 已完成）

**产出**
- 全库 22 目录 + 总览/资源/配色/架构/代码模板。
- 12 个 MVP 图表卡片（draft）+ 各族索引。
- 网络优秀示例（每图 ≥2）+ 来源台账。
- Framingham 测试区（数据说明/变量字典/任务清单/R&Py 测试脚本模板）。
- Skill 设计 5 文件。

**DoD**：结构完整、卡片可读、链接互通、规则可执行。✅

## 阶段 1：跑通 Framingham 最小集 ⏳（下一步）

**产出**
- 配好 R + Python 环境。
- 跑 [[可测试图表任务清单]] 的 8 个 ⭐ 任务（双后端）。
- 输出图存档 + 回填卡片第 9.2 节。

**DoD**：8 个 MVP 图 R/Python 都出图、过质检、卡片状态升 `tested`。

**阻塞**：运行环境；SEX/AGE_group 编码确认（见 [[测试结论与返修记录]] §5）。

## 阶段 2：知识 → 机器可读 spec ⬜

**产出**
- 把卡片 frontmatter + 代码模板 + `# >>> PARAM` 抽成 [[输入输出规范]] 的 `chart_spec`。
- 配色 registry 形式化（名 → 后端调用）。

**DoD**：能用一个 spec 文件驱动模板渲染出代码。

## 阶段 3：选型器 + 后端路由 + 代码生成 ⬜

**产出**
- 实现 [[图表决策流程]] 决策树。
- 实现 [[R与Python后端选择规则]] 路由。
- 模板渲染器：spec + 参数 → 可运行 .R/.py。

**DoD**：给「数据+问题+用途」自动产出选型理由 + 可运行代码。

## 阶段 4：质检 + 自动返修 ⬜

**产出**
- 实现 [[科研图表质量检查清单]] 打分器。
- 实现 [[图表返修规则总表]] 的 IF→THEN 后处理。

**DoD**：出图后自动评分、对常见问题自动改参重画、记录迭代。

## 阶段 5：回归测试 + 扩展 ⬜

**产出**
- 用 Framingham 任务清单做回归测试集，防止迭代回退。
- 扩展图表族：直方/密度/柱状/PCA/UMAP/火山/流程图等。
- 接更多数据集验证泛化。

**DoD**：新增/修改卡片后回归测试通过；图表族覆盖到第二批。

> 🟡 MVP = 已有可用最小实现（agent 驱动），待升级为确定性渲染器 / 自动化打分器。

## 优先级原则

1. **先深后广**：先把 12 MVP 图做到 `publication_ready`，再扩图类。
2. **测试驱动**：每图绑定 Framingham 任务，不跑通不算完成。
3. **接口先行**：阶段 2 的 spec 定好，后面自动化才顺。
4. **可复现优先**：任何阶段都产出代码而非黑盒图。
