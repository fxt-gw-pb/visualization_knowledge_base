# 图表决策流程

> 从「数据 + 研究问题」自动选图的决策树。这是 Skill 选型器的核心逻辑，也是人类选图的速查。落地在 [[图表类型索引]] 的分族上。

## 1. 顶层决策树（按研究问题）

```text
你想回答什么？
│
├─ 一个变量长什么样（分布）
│   ├─ 单变量频数 → 直方图 / 密度图
│   └─ 组间分布对比 → 箱线 / 小提琴 / 雨云
│         小样本→box+jitter / raincloud；要形态→violin；信息最全→raincloud
│
├─ 组之间有没有差异（比较）
│   ├─ 看整组分布 → 箱线 / 小提琴 / 雨云
│   ├─ 看均值+不确定性 → 点估计+CI
│   ├─ 看计数/比例总量 → 柱状（必从0）
│   └─ 看类别组成 → 堆叠 / 百分比堆叠
│
├─ 变量怎么联动（相关）
│   ├─ 两连续 → 散点(+回归/LOESS)；点多→hexbin
│   ├─ 多变量两两 → 相关矩阵热图 / pair plot
│   └─ 加第三/四维 → 颜色/大小（气泡）
│
├─ 随时间怎么变（趋势）
│   ├─ 组趋势+不确定性 → 折线 + CI带
│   ├─ 个体异质性 → spaghetti + 组均值
│   └─ 仅两时点 → 斜率图
│
├─ 模型说了什么（模型结果）
│   ├─ 多变量效应（OR/HR/RR）→ 森林图（log轴+参考线1）
│   └─ 线性系数 β → 系数图（参考线0）
│
├─ 模型准不准（预测评价）
│   ├─ 判别 → ROC（不平衡补PR）
│   ├─ 概率准确性 → 校准曲线
│   └─ 临床获益 → 决策曲线DCA
│
├─ 活多久/何时发病（生存）
│   └─ KM（+risk table+log-rank）；调整效应→Cox森林
│
└─ 很多变量压扁看（高维）
    ├─ 矩阵模式 → 热图（z-score/注释/聚类）
    └─ 全局结构 → PCA（线性）/ UMAP·tSNE（非线性）
```

## 2. 决策输入（机器可读判据）

| 判据 | 取值 | 影响 |
|---|---|---|
| x 类型 | continuous / categorical / ordered / time | 选族 |
| y 类型 | continuous / count / proportion / binary / time-event | 选图 |
| 变量个数 | 1 / 2 / 多 | 单变量分布 vs 关系 vs 高维 |
| 样本量 | 小(<30) / 中 / 大 | 是否叠点、KDE 是否可靠、是否 hexbin |
| 分组数 | 1 / 2-8 / >8 | 是否分面、颜色上限 |
| 是否有删失 | 是 / 否 | 生存图 vs 普通趋势 |
| 是否不平衡 | 是 / 否 | ROC 是否补 PR |
| 用途 | 论文 / 汇报 / 探索 | 尺寸/导出/信息密度 |

## 3. 歧义处理：给候选而非武断

当多个图等价适用，Skill 输出**主选 + 备选 + 理由**，引用卡片第 2/3 节（适用/不适用）：

```text
例：连续 × 二分类组，中等样本，看差异
主选： 箱线+jitter（[[箱线图_Boxplot]]）— 五数+原始点，最稳健
备选： 小提琴（[[小提琴图_Violin]]）— 若关心分布形态
备选： 雨云（[[雨云图_Raincloud]]）— 若想信息最全且样本不过大
不选： 柱状±SD — 隐藏分布，反模式
```

## 4. 选型后的下一步

选定图 → 进 [[Agent调用流程]] 第 3 步（后端路由）→ 取模板注入参数（[[输入输出规范]]）。

## 5. 反模式拦截（选型时直接挡掉）

| 想画 | 拦截理由 | 改 |
|---|---|---|
| 柱状±SD 表达连续均值 | 隐藏分布 | box/violin/point-range |
| 饼图 >5 类 | 难比较 | 柱状/棒棒糖 |
| 连线无序分类 | 假连续 | 柱状/点图 |
| rainbow 连续色 | 误导+不友好 | viridis |
| 截断柱状 y 轴 | 视觉撒谎 | 从0 / 换 point-range |

（细节见 [[图表返修规则总表]] 红线）
