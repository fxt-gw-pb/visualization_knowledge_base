# 输入输出规范

> 定义未来 Skill 的**输入契约**与**输出契约**，以及卡片如何提供机器可读的 spec。让“知识库 → 代码”可自动化。

## 1. 输入契约（用户/Agent → Skill）

```yaml
# plot_request.yaml
data:
  path: data_example/Framingham_data(1)_副本.csv
  filter: "PERIOD == 1"          # 可选：截面/子集
question: "比较有无高血压者的 BMI 差异"
intent: group_comparison          # distribution | comparison | correlation | trend | model | evaluation | survival | highdim
roles:                            # 变量角色（不确定可留 auto）
  group: PREVHYP
  value: BMI
purpose: paper                    # paper | talk | explore
backend: auto                     # auto | r | python
constraints:
  colorblind_safe: true
  max_colors: 8
  size: single_column             # single_column | double_column | slide
```

## 2. 中间 spec（卡片提供的机器可读结构）

每张卡片 frontmatter 已含核心字段，Skill 再补充“绘图 spec”（来自卡片第 5–8 节）：

```yaml
# chart_spec（由卡片渲染）
chart: boxplot
family: comparison
data_type: [continuous, categorical_group]
backend:
  r: {pkg: [ggplot2, ggpubr], template: "箱线图_Boxplot#7.4"}
  python: {pkg: [seaborn], template: "箱线图_Boxplot#8.4"}
params:                           # 对应模板里的 # >>> PARAM
  x: group
  y: value
  fill_palette: med_case_control  # registry 名，见 配色系统总览
  show_points_if_n_lt: 30
  add_pvalue: {global: kruskal, preset_pairs: true}
  y_from_zero: false
architecture:
  legend: auto                    # 两组可省
  size: single_column
repair_rules: ["xlabel_overlap->rotate", "small_n->add_points", "many_groups->facet_or_horizontal"]
quality_checklist: 箱线图_Boxplot#13
```

## 3. 输出契约（Skill → 用户）

```yaml
# plot_result
figure:
  files:
    - 07_示例图像库/Framingham测试图/fig_box_prevhyp_bmi_v1.pdf
    - 07_示例图像库/Framingham测试图/fig_box_prevhyp_bmi_v1.png
code:
  language: r                     # 实际用的后端
  script_path: scripts/fig_box_prevhyp_bmi.R
  reproducible: {seed: 2026, pkg_versions: {...}}
selection:
  chosen_chart: boxplot
  why: "连续结局 × 二分类组，中等样本，看分布差异"
  alternatives: [violin, raincloud, point_range]
quality:
  score: {A: 28, B: 22, C: 27, D: 14, total: 91}
  passed: true
repairs_applied: ["small_n->add_points: false（n大，改 alpha=0.2）"]
caveats: ["SEX 编码方向未确认（本图未涉及）"]
```

## 4. 命名与目录约定

| 产物 | 位置 | 命名 |
|---|---|---|
| 图 | `07_示例图像库/Framingham测试图/` 或用户指定 | `fig_<type>_<topic>_v<n>.<ext>`（[[R导出规范]]）|
| 代码 | `scripts/` | `fig_<type>_<topic>.<R\|py>` |
| 质检报告 | 随输出返回 | JSON/YAML |

## 5. 配色 registry 接口

Skill 不接受裸 hex，只接受 registry 名（见 [[配色系统总览]] §2）：

```text
分类： cat_main | cat_brewer_set2 | cat_npg
顺序： seq_viridis | seq_cividis | seq_blues | seq_batlow
发散： div_rdbu | div_vik | div_coolwarm
医学： med_case_control | med_treat_control | med_exposed | med_risk2
```
渲染时 registry → 后端具体调用（R `scale_*` / Python `palette`/`cmap`）。

## 6. 版本与状态门

- Skill 生产只用 `status: reviewed | tested | publication_ready` 的卡片。
- 每次出图把版本号写回卡片第 14 节“迭代记录”。
