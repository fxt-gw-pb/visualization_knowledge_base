# 科研数据可视化知识库总览

> 这是整个知识库的“地图”（Map of Content, MOC）。任何人或 Agent 进入本库，先读这一篇。

## 0. 这个库是什么

一个**长期可迭代**的科研数据可视化知识仓库，目标是支撑未来一个 **R / Python 双后端**的自动作图 Skill。

核心理念：

- 不在 R 与 Python 之间二选一，而是**按任务自动选择最佳后端**。
- 每一种图表都沉淀为**独立、结构统一、可被 AI 读取**的知识卡片。
- 不堆砌网络内容，而是**提炼成可执行规则**（适用/不适用、配色、架构、参数、返修）。
- 一切服务于一条主线：
  `知识仓库 → 图表规范 → 配色系统 → 图表模板 → 代码参数 → 示例图像 → Framingham 测试 → 可调用作图 Skill`

## 2. 三大横切系统

任何一张图都同时受三套规则约束，卡片里反复引用它们，避免重复造轮子：

1. **配色系统** → [[配色系统总览]]：按“分类 / 连续 / 发散 / 二分类”四种数据语义选色。
2. **架构系统** → [[图表架构总览]]：坐标轴、图例、标签、分面、尺寸、导出。
3. **质量与返修** → [[科研图表质量检查清单]] + [[图表返修规则总表]]：每张图收尾前必走。

## 3. 卡片状态机

每个图表卡片用 frontmatter 的 `status` 表示成熟度：

| status | 含义 | 进入条件 |
|---|---|---|
| `draft` | 初稿 | 完成基础整理（定位/场景/数据结构）|
| `reviewed` | 已审 | R/Python 思路、配色、架构、返修规则齐全且自检 |
| `tested` | 已测 | 用 Framingham 数据跑通，输出图存档 |
| `publication_ready` | 发表级 | 有高质量代码 + 示例图 + 完整返修与质检 |

## 4. 新增/迭代一张图的标准流程

```text
建立图表卡片
  ↓ 补充适用 / 不适用场景
  ↓ 检索网络优秀示例（记录来源 + license）
  ↓ 整理 R / Python 实现
  ↓ 写入配色与架构规则
  ↓ 用 Framingham 数据测试
  ↓ 输出示例图
  ↓ 执行质量检查（[[科研图表质量检查清单]]）
  ↓ 记录返修规则（[[图表返修规则总表]]）
  ↓ 更新 status
```

## 6. 给未来 Agent 的读取约定

- 每张卡片 frontmatter 的 `chart_name_en / chart_family / data_type / recommended_backend / status` 是**机器可读字段**，优先据此路由。
- 代码模板里凡是用 `# >>> PARAM` 标注的行，都是**可参数化注入点**。
- 配色不要写死十六进制，引用 [[配色系统总览]] 的命名色板（registry）。
