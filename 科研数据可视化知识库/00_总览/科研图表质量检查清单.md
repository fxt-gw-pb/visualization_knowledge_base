# 科研图表质量检查清单

> 每张图**收尾前必走一遍**。Agent 出图后逐项核对；任何一项不过，去 [[图表返修规则总表]] 找对应修法。
> 这是“通用清单”；每张卡片第 13 节还有该图**专属**的精简清单。

## A. 图表语义（图配不配数据）

- [ ] 图形类型匹配数据类型？（分类→柱/点；连续分布→箱/小提琴/密度；关系→散点/热图）
- [ ] 没有用**柱状图展示连续变量的分布**（应改 box/violin/raincloud）？
- [ ] 没有用**饼图展示过多类别**（>5 类别改柱状/棒棒糖）？
- [ ] 没有用**柱状图 + 误差棒**代替原始分布（"bar-bar plot" 反模式，改 box/violin/point-range）？
- [ ] 图是否清楚回答了**一个**研究问题（一图一结论）？

## B. 统计表达（数字说清楚没）

- [ ] 误差度量标注明确：是 **CI / SE / SD** 还是 IQR？图注里写清。
- [ ] p 值格式统一（见 [[标签与注释设计]]：`p < 0.001`，否则两位有效数字 `p = 0.023`）？
- [ ] 用了对数轴时**明确标注 log scale** 并给出真实刻度？
- [ ] 需要的 **reference line** 在场？（OR/HR 图的 x=1；差值图的 0；校准图的对角线；ROC 的对角线）
- [ ] 样本量 **n** 是否需要展示（分组 n、风险表 at-risk）？
- [ ] 多重比较是否合理（不是把所有两两 p 值都堆上去）？

## C. 视觉质量（好不好看、好不好读）

- [ ] 字体统一（全 figure 一种无衬线字体，如 Arial/Helvetica/Source Sans）？
- [ ] 字号适配用途（论文正文图轴标题 7–9pt；汇报图更大）？
- [ ] 坐标轴标签**不遮挡、不重叠**（必要时旋转 30–45° 或改横向）？
- [ ] 图例不过长、不喧宾夺主（能直接标注就别用图例）？
- [ ] 颜色**色盲友好**（过 Okabe-Ito / viridis 检验，见 [[色盲友好与打印友好原则]]）？
- [ ] 颜色数量受控（分类 ≤ 8–10；超了就分面/合并/加形状）？
- [ ] 多面板**间距、对齐、字号一致**，A/B/C/D 标签规范？
- [ ] 无意义装饰已去除（3D、阴影、渐变、网格过密、冗余边框）？
- [ ] 数据墨水比合理（Tufte：每一滴墨水都在传达数据）？

## D. 导出质量（能不能直接用）

- [ ] 论文图输出了 **PDF / SVG**（矢量）？
- [ ] PNG 至少 **300 dpi**（投稿线图常需 600+，含照片 300+）？
- [ ] 文件名规范（见 [[R导出规范]] / [[Python导出规范]]：`fig1_boxplot_chd_age_v2.pdf`）？
- [ ] 图像边界裁切正确（无多余白边/无截断，`bbox_inches='tight'` 或 ggsave 合适尺寸）？
- [ ] 背景符合用途（论文白底；叠在深色 slide 上可透明）？
- [ ] **保留了可复现代码**（脚本 + 数据版本 + 包版本）？

## E. 用途适配（最后一问）

- [ ] 论文用 → 单栏/双栏尺寸正确（见 [[论文单栏双栏尺寸]]）？
- [ ] 汇报用 → 字号放大、对比度足够、远处能看清（见 [[汇报型图表尺寸]]）？

## 评分口径（给 Agent 的量化版）

把 A–D 折成 0–100，便于自动判定是否进 `reviewed`：

```text
A 语义   30 分（任何一项硬伤直接降到 < 60，必须返修）
B 统计   25 分
C 视觉   30 分
D 导出   15 分

≥ 90  → 可标 reviewed / 接近 publication_ready
75–89 → 小修
< 75  → 进 [[图表返修规则总表]] 系统性返修
```
