# 分面与多面板设计

> 两件相关但不同的事：**分面（facet）**= 同一种图按某变量自动拆成小多图；**多面板（multi-panel）**= 把**不同**的图拼成一张 Figure（A/B/C/D）。

## 1. 分面（facet / small multiples）

### 何时分面

- 想比较**同一关系在不同子群里的样子**（如 BMI~年龄 分性别）。
- 类别多到一张图塞不下时（[[分类变量配色]] 超色上限的解法之一）。
- 强调“结构一致、数据不同”。

### 规则

- 所有子面**共享坐标轴范围**（`scales="fixed"`）便于比较；除非各子群量级差太大才放开（`free`），并明确说明。
- 子面数量适中（2×2、2×3、3×3）；太多改翻页或精选。
- strip 标签清楚（人类可读名）。
- 行列安排有逻辑（按时间/按严重度排序）。

```r
+ facet_wrap(~ AGE_group, scales = "fixed")          # >>> PARAM
+ facet_grid(SEX ~ PERIOD)
```
```python
g = sns.FacetGrid(df, col="AGE_group", row="SEX", sharex=True, sharey=True)
g.map_dataframe(sns.scatterplot, x="BMI", y="TOTCHOL")
```

## 2. 多面板（multi-panel figure）

### 何时拆成多面板而非分面

- 子图是**不同类型的图**（A 箱线、B 散点、C 森林、D KM）讲一个完整故事。
- Figure 1（baseline）/ Figure 2（model result）这种叙事结构。
- 详见 [[多面板组合图_MultiPanel]]。

### A/B/C/D 标签规范

- 左上角**加粗大写字母**（A、B、C…），位置一致（统一在每个子图左上）。
- 字号比轴标题大、与正文图注呼应。
- 工具：R patchwork `plot_annotation(tag_levels = 'A')`；Python `ax.text(-0.1,1.05,"A",transform=ax.transAxes,weight="bold")` 或 `fig.text`。

### 多面板一致性（必查）

- 所有子图**字体、字号、配色、线宽统一**。
- **共享图例合并到一处**（别每个子图各放一个相同图例）。
- 子图间距均匀、对齐（坐标轴对齐用 cowplot/patchwork）。
- 子图大小按信息量分配（重要的大一点）。

## 3. 何时分面 vs 拆图 vs 不拆

```text
同种图、看子群差异        → 分面 facet
不同种图、讲一个故事      → 多面板 multi-panel（A/B/C/D）
组别太多、单图挤          → 先分面；仍挤就拆成多图/补充材料
两三组而已                → 别拆，叠在一张图 + 直接标注
```

## 4. 工具选择

| 需求 | R | Python |
|---|---|---|
| 同种图分面 | `facet_wrap/grid` | `FacetGrid` / `relplot(col=...)` |
| 不同图拼版（随手）| patchwork `p1+p2` `/` | `plt.subplots` / `subplot_mosaic` |
| 严格对齐坐标轴 | cowplot `plot_grid(align="v")` | `gridspec` + 手动对齐 |
| 自动 A/B/C/D | patchwork `tag_levels` | 手动 `ax.text` |
| 共享图例 | cowplot `get_legend` / patchwork `guides="collect"` | 一个 `fig.legend` |

详见 [[patchwork与cowplot组合图]] 与 [[多面板组合图_MultiPanel]]。

## 5. 常见错误（→ 返修）

- ❌ 多面板子图字号/配色不一致。
- ❌ 每个子图重复同一个图例 → 合并到一处。
- ❌ 分面用了 `free` 轴还拿来横向比较 → 误导。
- ❌ A/B/C/D 位置/字号不统一。
- ❌ 子图对齐差（坐标轴错位）。
