# 论文单栏双栏尺寸

> 论文图要**按最终印刷尺寸设计**（所见即所得），不要先画大再缩小——缩小会让字号失控、线宽消失。
> 具体阈值各刊不同，**投稿前查目标期刊的 Figure/Artwork guidelines**；下面是通用经验值。

## 1. 常用宽度（经验值）

| 类型 | 宽度（mm）| 宽度（inch）| 说明 |
|---|---|---|---|
| 单栏 single column | 85–90 | 3.3–3.5 | 大多数期刊单栏 ~3.5 in |
| 1.5 栏 | 110–140 | 4.3–5.5 | 部分期刊允许 |
| 双栏 double / full width | 170–180 | 6.7–7.0 | 跨整页宽 |
| 高度 | ≤ ~230 | ≤ ~9 | 不超过版心高度，留图注空间 |

> Nature 系常见：单栏 89 mm，双栏 183 mm，最大高度约 247 mm。各刊以官方为准。

## 2. 字号 / 线宽（终稿尺寸下）

| 元素 | 建议 |
|---|---|
| 轴标题 | 7–9 pt |
| 刻度标签 | 6–8 pt |
| 面板标号 A/B/C | 8–10 pt 加粗 |
| 线宽 | ≥ 0.5–1 pt（缩放后仍可见）|
| 点大小 | 适中，避免过小消失或过大糊成一团 |

字体：无衬线（Arial/Helvetica/Source Sans/Nimbus Sans），全文统一。

## 3. 设计要点

1. **按目标尺寸出图**，`ggsave(width=, height=, units="mm")` / `figsize=(inch)`，不要事后缩放。
2. 单栏图信息别太满（窄），双栏图可放多面板。
3. 终稿尺寸下肉眼检查：字看得清吗？线还在吗？点糊了吗？
4. 矢量优先：PDF/SVG/EPS，字体嵌入（[[R导出规范]] / [[Python导出规范]]）。

## 4. 补充材料（Supplementary figure）

- 尺寸限制宽松，可更大/更多面板。
- 但仍保持与正文图一致的字体/配色/风格。
- 命名清楚：`figS1_...`。

## 5. 代码片段

**R**

```r
# 单栏 89mm × 70mm
ggsave("fig1.pdf", p, width = 89, height = 70, units = "mm", device = cairo_pdf)
# 双栏 183mm
ggsave("fig2.pdf", p, width = 183, height = 120, units = "mm", device = cairo_pdf)
```

**Python**

```python
mm = 1/25.4
fig, ax = plt.subplots(figsize=(89*mm, 70*mm))   # 单栏
fig.savefig("fig1.pdf", bbox_inches="tight")     # 矢量
# 双栏：figsize=(183*mm, 120*mm)
```

## 6. 常见错误（→ 返修）

- ❌ 大图缩小投稿 → 字号/线宽失控，改为按终稿尺寸出图。
- ❌ 单栏塞太多内容 → 拆面板或转双栏/补充材料。
- ❌ 字号过小（终稿下 < 6 pt 难读）。
- ❌ 线宽太细缩放后消失。
- ❌ 不同图字体不一致。
