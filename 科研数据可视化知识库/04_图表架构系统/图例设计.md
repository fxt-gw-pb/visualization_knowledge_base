# 图例设计

> 图例是“颜色/形状 → 含义”的查找表。**最好的图例是没有图例**——能直接标注就别让读者来回查表。

## 1. 什么时候需要图例

| 情况 | 要图例吗 |
|---|---|
| 多组用颜色/形状区分，且组多 | 要 |
| 只有 2–3 条线/组 | **优先直接标注**（direct labeling），不用图例 |
| 连续色编码 | 要 **colorbar**（带刻度与单位）|
| 单一系列 | 不要 |
| 分面已经在 strip 标了组 | 通常不再需要图例 |

## 2. 直接标注优先（direct labeling）

- 折线图：把组名标在**线的末端**（[[折线趋势图_Line]]）。
- 散点分组：少组时在簇旁标文字。
- 工具：R `ggrepel::geom_text_repel`、`directlabels`；Python `ax.annotate` / `ax.text`。
- 好处：减少眼动、省空间、色盲也能读（文字是第二通道）。

## 3. 图例位置规则

| 位置 | 何时 |
|---|---|
| 图内空白角落 | 有空白且不挡数据（散点/折线常用右上/右下）|
| 右侧 | 类别多、需要垂直列表 |
| 底部（横排/多列）| 宽图、上下空间紧、多类别 → 也利于论文双栏 |
| 顶部 | 类别少、想先声明分组 |

- 图例**不应遮挡数据**；挡到了就移位或缩小。
- 图例过长 → 移底部分多列 / 拆分 / 改直接标注（返修规则）。

## 4. 图例内容规范

- 图例标题用人类可读名 + 必要单位（`Hypertension`，不是 `PREVHYP`）。
- 类别顺序与数据/语义一致（如 No→Yes、Low→High），别乱序。
- 连续 colorbar 要有**刻度 + 单位 + 合理范围**（截断离群，见 [[连续变量配色]]）。
- 删掉无意义的图例标题（如 ggplot 默认把变量名当标题时改掉或去掉）。

## 5. 代码片段

**R / ggplot2**

```r
+ theme(legend.position = "bottom") +          # >>> PARAM 位置
  guides(fill = guide_legend(nrow = 1, title = "Hypertension"))
# 直接标注线末端：
library(ggrepel)
+ geom_text_repel(data = ends, aes(label = group), nudge_x = 0.3, direction = "y")
# 去掉图例： + theme(legend.position = "none")
```

**Python / matplotlib**

```python
ax.legend(loc="lower right", frameon=False, ncol=1, title="Hypertension")  # >>> PARAM
# 底部居中、多列：
ax.legend(loc="upper center", bbox_to_anchor=(0.5,-0.12), ncol=3, frameon=False)
# 直接标注：
ax.annotate("Smokers", xy=(x_end,y_end), xytext=(5,0), textcoords="offset points")
```

## 6. 常见错误（→ 返修）

- ❌ 2–3 组还非要图例，读者反复查表 → 直接标注。
- ❌ 图例盖住数据点。
- ❌ 图例标题还是变量名 `PREVHYP`。
- ❌ 连续 colorbar 没单位/被离群值拉爆。
- ❌ 图例顺序与图中顺序不一致。
