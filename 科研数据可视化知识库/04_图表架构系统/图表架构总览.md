# 图表架构总览

> “架构”= 一张图里**非数据的结构元素**怎么安排：坐标轴、图例、标签注释、分面、尺寸、导出。
> 配色管“用什么颜色”（[[配色系统总览]]），架构管“怎么摆放与表达”。两者加图表卡片，构成一张图的完整规范。

## 1. 一张图的架构清单（从外到内）

```text
画布尺寸 / 长宽比      → [[论文单栏双栏尺寸]] / [[汇报型图表尺寸]]
└ 坐标轴（范围/刻度/单位/log/截断/参考线） → [[坐标轴设计]]
  └ 几何对象（图表卡片第 5 节定）
    └ 图例（位置/是否需要/直接标注）       → [[图例设计]]
      └ 标签与注释（p 值/CI/重点结论/n）    → [[标签与注释设计]]
        └ 分面 / 多面板（A/B/C/D）          → [[分面与多面板设计]]
          └ 导出（PDF/SVG/PNG dpi/命名）    → [[R导出规范]] / [[Python导出规范]]
```

## 2. 架构设计的总原则

1. **数据墨水比优先（Tufte）**：每个非数据元素都要问“删了会不会损失信息”，不会就删。
2. **位置 > 颜色 > 形状/大小**：人对位置最敏感，关键信息优先用位置编码。
3. **一图一结论**：架构服务于这张图要讲的那**一句话**。
4. **可读性按用途定**：论文（紧凑、矢量、小字号）vs 汇报（放大、对比强）走不同尺寸表。
5. **一致性**：同一 figure / 同一文里，字体、字号、配色、面板标号风格统一。

## 5. 默认值（Agent 缺省）

| 元素 | 论文缺省 | 汇报缺省 |
|---|---|---|
| 字体 | 无衬线（Arial/Helvetica/Source Sans）| 同 |
| 轴标题字号 | 8–9 pt | 16–20 pt |
| 刻度字号 | 7–8 pt | 14–18 pt |
| 线宽 | 0.5–1 pt | 1.5–2.5 pt |
| 图例 | 右侧或图内直接标注 | 底部，字大 |
| 背景 | 白底、无重网格 | 白底（深色 slide 可透明）|
| 导出 | PDF/SVG + 必要时 PNG 600dpi | PNG 300dpi + PDF |
